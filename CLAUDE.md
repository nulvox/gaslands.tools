# Gaslands.tools — Gaslands Team Roster Builder

## Project Overview

A browser-based, fully interactive team roster builder and manager for Gaslands Refuelled. Players create teams by selecting a sponsor, setting a budget, adding vehicles, and equipping them with weapons, upgrades, and perks — all with real-time cost tracking and rules validation. The tool can export styled standalone HTML roster sheets, JSON data, and shareable URLs. Built with Go compiled to WebAssembly for core logic, with a vanilla HTML/CSS/JS frontend. Hosted as static files on GitHub Pages.

## Tech Stack

- **Core logic**: Go 1.23+, compiled to `GOOS=js GOARCH=wasm`
- **Frontend**: Vanilla HTML5, CSS3, JavaScript (no framework)
- **Storage**: `localStorage` (JSON-serialized team data)
- **PDF Extraction**: Python (uv-managed) with pdfplumber for extracting game data from the rulebook PDF
- **Print/PDF Export**: Generate styled HTML export view, trigger browser print dialog
- **Build**: Makefile; output is a `docs/` directory for GitHub Pages
- **Testing**: Go standard `testing` package
- **Linting**: `golangci-lint` for Go; browser-console-clean JS

## Architecture

Go WASM serves as the logic backend. It exposes functions to JavaScript via `syscall/js`:

- Team CRUD (create, read, update, delete)
- Vehicle management (add/remove/configure vehicles within a team)
- Game data queries (vehicle types, weapons, upgrades, perks, sponsors from built-in database)
- Validation (budget limits, build slot usage, sponsor perk class restrictions)
- JSON import/export (serialize/deserialize full team state)
- HTML export (generate styled standalone HTML roster page via Go templates)

The JS frontend handles all DOM manipulation, event binding, tab management, and layout. It calls into Go WASM for all data operations — the JS side never directly mutates team data.

### Three-Layer Separation

```
internal/gamedata/   — Pure Go. Built-in database of all game items. No syscall/js.
internal/team/       — Pure Go. Team/vehicle model, validation, serialization, HTML export. No syscall/js.
cmd/wasm/main.go     — Thin bridge. Imports syscall/js. Marshals between JS strings and Go structs.
web/js/              — Frontend. DOM rendering, events, localStorage, calls WASM functions.
```

### Data Flow

```
User selects weapon from dropdown
  -> JS event handler captures selection
  -> JS calls WASM function (e.g., addWeapon(teamJSON, vehicleID, weaponName))
  -> Go looks up weapon in gamedata, adds to vehicle, validates team
  -> Go returns {team: json, warnings: []}
  -> JS updates DOM with updated vehicle card
  -> JS persists to localStorage
```

## Team Roster Data Model

### Team (top-level entity, one per tab)

- **ID**: string (UUID)
- **Name**: string (team name, e.g., "THE WARDEN")
- **Sponsor**: SponsorSelection — reference to sponsor from database, or custom
- **Budget**: int (default 50 Cans, user-configurable)
- **Vehicles**: ordered list of Vehicle
- **Notes**: string (freeform — strategy, tactics, lore, anything)
- **Version**: string (JSON schema version, starts at "1.0")

### Vehicle

- **ID**: string (UUID)
- **CustomName**: string (user-assigned, e.g., "IRON COFFIN #1")
- **Role**: string (freeform tag, e.g., "ANCHOR / BRAWLER")
- **VehicleType**: string (key into vehicle type database, e.g., "car", "truck")
- **Variant**: string (optional modifier, e.g., "Prison Car")
- **Weapons**: ordered list of WeaponInstance
- **Upgrades**: ordered list of UpgradeInstance
- **Perks**: ordered list of PerkInstance
- **Notes**: string (freeform per-vehicle notes)

### WeaponInstance

- **Name**: string
- **Cost**: int (Cans)
- **AttackDice**: string (e.g., "3D6")
- **Range**: string (e.g., "Double", "Medium")
- **Slots**: int (build slots consumed)
- **SpecialRules**: string
- **IsCustom**: bool (true = user-created, false = from database)

### UpgradeInstance

- **Name**: string
- **Cost**: int (Cans)
- **Slots**: int (build slots consumed)
- **Description**: string
- **IsCustom**: bool

### PerkInstance

- **Name**: string
- **Cost**: int (Cans)
- **Class**: string (perk class, e.g., "Aggression", "Badass")
- **Description**: string
- **IsCustom**: bool

### SponsorSelection

- **Name**: string (sponsor name from database, or custom)
- **IsCustom**: bool
- If custom: user provides PerkClasses []string and SponsorPerks manually

## Built-in Game Database

The `internal/gamedata/` package contains hardcoded data extracted from the Gaslands Refuelled rulebook PDF. This data is extracted by `tools/extract-pdf/extract.py` into `data/*.md` files, then manually reviewed and transcribed into Go source.

### Vehicle Types

Each entry: Name, WeightClass (Lightweight/Middleweight/Heavyweight), BaseCost (Cans), Hull, Handling, MaxGear, Crew, BuildSlots, SpecialRules

Examples from the rulebook: Bike, Buggy, Car, Performance Car, Pickup Truck, Truck, Bus, Monster Truck, War Rig, Gyrocopter, Helicopter, Tank, etc.

### Weapons

Each entry: Name, Cost (Cans), AttackDice, Range, Slots, SpecialRules, MountRestrictions

Examples: Handgun (free, crew weapon), Machine Gun, Heavy Machine Gun, Minigun, Rockets, Flamethrower, Mine Dropper, Oil Slick Dropper, etc.

### Upgrades

Each entry: Name, Cost (Cans), Slots, Description, Restrictions

Examples: Ram, Exploding Ram, Armour Plating, Roll Cage, Nitro Booster, Tank Tracks, etc.

### Perks

Each entry: Name, Cost (Cans), Class, Description

Classes include: Aggression, Badass, Speed, Precision, Military, Technology, etc.

Examples: Battlehammer (Aggression, 4 Cans), Grinderman (Aggression, 5 Cans), Road Warrior (Badass), etc.

### Sponsors

Each entry: Name, FlavorText, AllowedPerkClasses []string, SponsorPerks []SponsorPerk

SponsorPerk: Name, Description, Effect

Examples: The Warden, Rutherford, Miyazaki, Scarlett Annie, Idris, Highway Patrol, Maxxine, Mishkin, Slime, Verney, Bevill, etc.

### Custom Entries

For every category (vehicles, weapons, upgrades, perks), users can create custom entries by selecting "Custom..." from the database dropdown. Custom entries have IsCustom=true and all fields are user-editable. This supports homebrew content and house rules.

## Validation Rules (Soft Warnings Only)

All validation produces warning messages, never blocks saves:

- **Budget**: Total cost of all vehicles (base cost + weapons + upgrades + perks, adjusted by variant/sponsor discounts) should not exceed team budget. Warn if over, show exact overage.
- **Build Slots**: Total slots consumed by weapons + upgrades on a vehicle should not exceed the vehicle type's build slots. Warn per vehicle.
- **Sponsor Perk Classes**: Perks should belong to the team's sponsor's allowed perk classes. Warn if a perk's class is not in the sponsor's list. (Does not apply to custom sponsors.)
- **Vehicle Variants**: Some variants have weight class or vehicle type restrictions (e.g., Prison Car is Middleweight only). Warn if requirements not met.
- **Empty Team**: Warn if team has zero vehicles.

## Key UX Requirements

- **Tabs**: Each team roster opens in a tab. Tabs can be created, closed, renamed.
- **Auto-save**: Changes persist to localStorage immediately on edit.
- **Vehicle Cards**: Each vehicle renders as a card showing stats, weapons, upgrades, perks, cost breakdown, and slot usage bar.
- **Database Dropdowns**: When adding vehicles/weapons/upgrades/perks, show a dropdown of items from the built-in database. Include a "Custom..." option for freeform entries.
- **Budget Summary Bar**: Always-visible bar showing: Budget, Spent, Remaining, Vehicle Count, Total Hull.
- **JSON Export**: Download team as `.gaslands.json` file.
- **JSON Import**: Upload `.json` file to load a team.
- **HTML Export**: Generate a standalone styled HTML file (like `examples/the-warden-roster.html`) and trigger download.
- **Print**: Render the HTML export view and trigger browser print dialog for print-to-PDF.
- **URL Sharing**: Encode team data in URL hash for shareable links.
- **Theme**: Post-apocalyptic visual theme (rust red, amber, soot black). Dark mode toggle.
- **Responsive**: Desktop-first (1024px+). Mobile is nice-to-have.

## HTML Export Reference

See `examples/the-warden-roster.html` for the reference implementation of the HTML export. Key design elements:

- **Fonts**: Black Ops One (titles), Share Tech Mono (data), Oswald (accents) via Google Fonts CDN
- **Colors**: Rust red (#c0392b), amber (#f39c12), soot black (#1a1208), green (#27ae60), blue (#2980b9)
- **Layout**: Header with sponsor/budget info, tabbed content (Roster, Sponsor, Strategy, Key Rules)
- **Vehicle Cards**: Stat grids, weapon/upgrade/perk lists with costs, cost breakdown tables, slot usage bars
- **Budget Summary**: Pill-style stat badges
- **Single-file**: All CSS inline, no external dependencies except Google Fonts CDN links
- **JavaScript**: Minimal — only tab switching

The HTML export is generated by Go's `html/template` package in `internal/team/export.go`. The template is embedded in the Go binary.

## Gaslands Rules Quick Reference

Key concepts that affect the roster builder:

- **Cans**: Currency for team building. Common budgets: 20 (starter), 50 (standard), 100 (large), 150 (epic).
- **Weight Classes**: Lightweight (bikes, buggies), Middleweight (cars, pickups, trucks), Heavyweight (buses, war rigs, tanks). Affects base stats and available modifications.
- **Build Slots**: Each vehicle has a fixed number. Weapons and some upgrades consume slots. Cannot exceed maximum.
- **Sponsors**: Factions granting access to specific perk classes and unique sponsor abilities. One sponsor per team.
- **Perk Classes**: Aggression, Badass, Speed, Precision, Military, Technology, etc. Each sponsor allows 2-3 classes.
- **Weapons**: Range from free crew weapons (Handgun) to expensive mounted weapons. Cost Cans and build slots.
- **Upgrades**: Vehicle modifications. Cost Cans and may cost build slots.
- **Perks**: Driver/crew abilities. Cost Cans but typically don't cost build slots.
- **Variants**: Vehicle modifiers (e.g., Prison Car reduces cost and hull). Applied per-vehicle, may have weight class restrictions.
- **Audience Votes**: Earned during play, not tracked in roster builder.
- **Hazard Tokens**: Gameplay mechanic, not tracked in roster builder.

Full game data is extracted from the rulebook PDF by `tools/extract-pdf/extract.py` and hardcoded into `internal/gamedata/`.
